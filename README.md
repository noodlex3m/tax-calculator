> **[UA]** Повноцінний інтерактивний веб-сервіс для ФОП на спрощеній (1, 2, 3 групи) та загальній системі оподаткування. Сервіс інтегрує актуальні ліміти та ставки ПКУ на 2026 рік, розумний пошук КВЕДів з логікою обмежень, хмарний довідник FAQ, динамічний податковий вісник та кастомну CMS-панель керування з розширеними правами адміністрування.
>
> **[EN]** Advanced, serverless-driven tax calculation & content management platform for Ukrainian entrepreneurs (1st, 2nd, 3rd groups & general taxation system) updated for 2026 fiscal regulations. Features a fully custom markdown/HTML CMS editor, real-time comment moderation, and automated tax compliance tools.

🔗 **Офіційний сайт:** [tax.serh.one](https://tax.serh.one/)

## 🚀 Основний функціонал (Key Features)

Проєкт пропонує комплексні продуктові та інженерні рішення:

### 🧮 1. Розрахунок податків 2026 (UI/UX 2.0)
*   **Модернізований інтерфейс Dashboard-типу:** Повний перехід від класичних fieldset-форм до сучасного двоколонкового макету (введення параметрів ліворуч, результати та графік — праворуч). Впроваджено сегментовані вкладки (Tabs) для вибору системи, інтерактивні картки груп ФОП та плавний повзунок (Range Slider) річного доходу, обмежений лімітом обраної групи.
*   **Точний прорахунок загальної системи:** Розрахунок ПДФО (18%), Військового збору (5%) та ЄСВ (22% з чистого прибутку, що обмежений мінімальним 1902.34 ₴ та максимальним 38 046.80 ₴ місячними лімітами).
*   **Інтеграція пільг ЄСВ та Облікової картки:** Автоматичне завантаження пільги ЄСВ з профілю користувача у Firestore та повне обнулення ЄСВ при виборі пільг (пенсіонери, особи з інвалідністю, працевлаштовані, мобілізовані) з виведенням інформаційних підказок про законодавчі нюанси.
*   **Аналітична візуалізація та адаптивна легенда:** Побудова кругової діаграми розподілу та прогрес-бару часток бюджету (чистий прибуток / податки / ЄСВ / військовий збір). Легенду переведено на адаптивну HTML-сітку, що вирішило проблему з обрізанням військового збору на мобільних.
*   **Розумні сповіщення:** Системні попередження про перевищення лімітів доходу для обраної групи, обов'язкову реєстрацію ПДВ та інші законодавчі обмеження.

### 🔍 2. База знань та Податковий вісник (Firestore Backend)
*   **Динамічні хмари:** Повний перехід від локальних статичних JSX-масивів до хмарних NoSQL колекцій у **Firebase Firestore** для новин та довідника FAQ.
*   **Оптимальне читання (Performance):** Використання оптимізованих запитів `getDocs` з локальним сортуванням на клієнті, що усуває зайві фонові WebSocket-з'єднання, гарантує миттєву швидкість завантаження сторінки та зберігає ліміти безкоштовного тарифу **Firebase Spark Plan**.
*   **Автоматичний Fallback (UX-страховка):** Якщо хмарне з'єднання відсутнє або база даних порожня, система автоматично підвантажує вбудовану офлайн-базу запитань, запобігаючи білому екрану.

### 🛡️ 3. Панель керування адміністратора (CMS)
*   **Захищені маршрути:** Роут `/admin` надійно прихований клієнтським фільтром `ProtectedRoute` (авторизованих користувачів без ролі адміна зустрічає плашка "Доступ обмежено", а неавторизовані — автоматично перенаправляються на сторінку `/login`).
*   **Дворежимний редактор коду (Quill & HTML IDE):** 
    *   **✍️ Редактор:** Візуальний WYSIWYG-інтерфейс на базі `react-quill-new` для красивого форматування.
    *   **💻 HTML Код:** Кастомний моноширинний редактор коду в темних тонах (IDE-style), що дозволяє редагувати HTML-розмітку напряму.
    *   *Двосторонній зв'язок (two-way binding):* Обидва режими синхронізуються дзеркально в реальному часі — міняєш HTML, перемикаєшся в редактор, бачиш результат!
*   **Динамічні категорії без хардкоду:** Можливість створювати нові категорії для новин чи FAQ безпосередньо через інтерфейс адмінки. Фронтенд самостійно відсканує базу даних і автоматично згенерує красиві вкладки-чіпси для фільтрації на клієнті.

### 💬 4. Інтерактивна стрічка коментарів та бан-система
*   **Гнучке обговорення:** Деревовидна структура коментарів (відповіді на повідомлення), лайки/дизлайки та сортування за часом/популярністю.
*   **Клієнтська премодерація:** Автоматичне блокування нецензурної лексики перед відправкою.
*   **Адміністрування в реальном часі:** Можливість бану та розбану користувачів в один клік прямо зі стрічки новин.
*   **Серверний захист (Firestore Security Rules):** Заблоковані користувачі миттєво відсікаються базою даних на рівні правил безпеки, а інтерфейс для них замінюється на попереджувальний банер.

### 💼 5. Державний реєстр КВЕД
*   **Повний довідник:** Інтегровано повний реєстр КВЕД України (996 елементів).
*   **Бізнес-логіка ПКУ:** Кожен КВЕД містить прописані правила заборон та обмежень згідно зі ст. 291 Податкового кодексу України (наприклад, заборона 2-ї групи для телекомунікацій чи аудіовізуальних послуг).

### 📇 6. Професійна Облікова картка ФОП (ПКУ-сумісна)
*   **Реактивна податкова логіка:** Автоматична валідація полів згідно з ПКУ. Вибір системи оподаткування чи групи реактивно блокує або налаштовує інші параметри (наприклад, для 1-2 груп фіксується пільгова ставка та вимикається ПДВ, а для 3 групи ставка 3% блокує вибір ПДВ на "Так", 5% — на "Ні").
*   **Інтерактивний КВЕД-реєстр:** Пошук у реальному часі за базою КВЕДів з можливістю додавати декілька видів діяльності та призначити один з них основним.
*   **Кількісний облік РРО/ПРРО:** Окремий облік кількості класичних РРО та програмних ПРРО.
*   **Реєстр 20-ОПП (451 тип об'єктів):** Інтегровано повний офіційний довідник об'єктів оподаткування ДПС України. Користувач може шукати типи об'єктів за кодом/назвою та створювати власні картки із зазначенням назв та адрес.
*   **Пільги ЄСВ:** Можливість вказати законні підстави звільнення від сплати ЄСВ (пенсіонери, особи з інвалідністю, офіційно працевлаштовані, мобілізовані).
*   **Кількість працівників:** Окреме поле для обліку найманих працівників з автоматичними обмеженнями згідно з ПКУ (заборона для 1-ї групи, макс. 10 для 2-ї групи ФОП, без обмежень для інших).
*   **Автоматична міграція та сумісність:** Профіль ФОП автоматично мігрує старі текстові дані у хмарі до нової глибокої структури та назад, усуваючи ризик порушити роботу старих профілів.

### 👤 7. Панель керування ФОП та Індивідуальні Календарі (CRM)
*   **Консоль управління користувачами:** Адміністратор бачить список усіх зареєстрованих користувачів (інформація автоматично синхронізується з колекцією `users` при вході/реєстрації).
*   **Створення ФОП без email (Офлайн-режим):** Адміністратор може вести картки ФОП для друзів чи клієнтів, які не зареєстровані в системі, залишаючи поле email порожнім. Це дозволяє використовувати калькулятор як зручний особистий органайзер.
*   **«Лінива» прив'язка (Lazy Linking):** Якщо адміністратор створив картку ФОП з електронною поштою клієнта, який ще не зареєстрований, система автоматично зв'яже цю картку з його UID під час першого ж входу користувача з цим email. Тимчасовий гостьовий документ при цьому автоматично та безпечно видаляється.
*   **Ручна прив'язка акаунтів:** Можливість в один клік зв'язати будь-яку офлайн-картку ФОП із будь-яким зареєстрованим користувачем системи через зручний випадаючий список у вікні редагування.
*   **Керування календарем дедлайнів:** Адміністратор може переглядати та відмічати виконані податкові завдання/дедлайни безпосередньо для кожної картки ФОП клієнта, не впливаючи на власні завдання. Працює на базі інтегрованого компонента `TaxCalendar` у режимі делегування.

### 📢 8. Інтеграція з Telegram та Telegram Mini App
*   **Автоматичний автопостинг (Telegram Bot API):** Впроваджено функціонал публікації новин та корисних Q&A (FAQ) з адмін-панелі сайту безпосередньо у ваш Telegram-канал `@taxuse`. Повідомлення автоматично екрануються за правилами `MarkdownV2` та містять активні посилання на сторінки сайту.
*   **Telegram Mini App (TWA):** Проєкт налаштовано як Telegram Mini App, що дозволяє користувачам працювати з калькулятором податків безпосередньо в месенджері за посиланням `t.me/TaxSerhOneBot/calc` із повною підтримкою адаптивності та темної теми.

## 🛠 Технологічний стек (Tech Stack)

### Frontend-ядро
*   **Framework:** React 19 (Hooks, Context API, Code Splitting через `lazy` & `Suspense`)
*   **Routing:** React Router DOM v7
*   **SEO & Meta:** Вбудований механізм React 19 Document Metadata (нативне динамічне керування унікальними тегами `title` / `meta description` для кожної сторінки без використання сторонніх бібліотек)
*   **Visuals:** Chart.js + React-Chartjs-2
*   **Rich Text Editor:** `react-quill-new` (професійна адаптація під React 19)
*   **Styling:** Vanilla CSS з нативними CSS-змінними (сучасний неоморфний/скляний інтерфейс з повною підтримкою **Темної теми**)
*   **PWA:** Service Workers з плагіном `Vite PWA` (додаток повністю автономний і встановлюється на iOS/Android/ПК)

### Cloud-сервіси та безпека
*   **Authentication:** Firebase Auth (керування сесіями, безпечна реєстрація та авторизація)
*   **Database:** Firebase Firestore (хмарна NoSQL база даних)
*   **Security:** Firestore Security Rules (серверна ізоляція розрахунків користувачів, рольова модель доступу для адміна через пошту `noodlex3m@gmail.com` та автоматичне блокування порушників)
*   **Email Service:** EmailJS (Serverless відправка повідомлень зворотного зв'язку на пошту розробника без утримання Node.js-бекенду)
*   **CI/CD:** Netlify (автоматичний деплой при пушах у GitHub)

## 💻 Як запустити проєкт локально (Installation)

1.  **Клонуйте репозиторій:**
    ```bash
    git clone https://github.com/noodlex3m/tax-calculator.git
    cd tax-calculator
    ```

2.  **Встановіть залежності:**
    ```bash
    npm install
    ```

3.  **Налаштуйте змінні середовища:**
    Створіть файл `.env` у корені проєкту та вставте ваші конфігураційні ключі:
    ```env
    # Firebase Configuration
    VITE_FIREBASE_API_KEY=your_api_key
    VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
    VITE_FIREBASE_PROJECT_ID=your_project_id
    VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
    VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
    VITE_FIREBASE_APP_ID=your_app_id
    
    # EmailJS Configuration
    VITE_SERVICE_ID=your_service_id
    VITE_TEMPLATE_ID=your_template_id
    VITE_PUBLIC_KEY=your_public_key
    ```

4.  **Команди розробки та валідації:**
    *   **Запуск сервера розробки (Vite):** `npm run dev`
    *   **Збірка для продакшну (Production build):** `npm run build`
    *   **Запуск лінтера ESLint:** `npm run lint`
    *   **Запуск Vitest тестів:** `npm test`
